Business Process Manager Integration Service 模式

可以使用此模式将 IBM Business Process Manager (BPM) 集成服务和 IBM Integration Bus 服务集成在一起。

解决方案

可以使用此模式生成可由 IBM Business Process Manager 调用的新 IBM Integration Bus 服务。

在使用此模式之前，请阅读 IBM Integration Bus 信息中心中关于 IBM Business Process Manager 的信息；请参见使用 Business Process Manager

BPM 和 IBM Integration Bus 开发人员合作，通过完成下列步骤将 IBM BPM 集成服务与 IBM Integration Bus 服务集成在一起：

首先，BPM 开发人员必须完成下列任务：

  1. 使用 BPM 工具箱创建或标识需要调用 IBM Integration Bus 服务的 BPM 集成服务。
  2. 创建一个 BPM 导出（.twx 文件），其中包含一个或多个定义了输入和输出变量的集成服务。

下一步，IBM Integration Bus 开发人员必须完成下列任务：

  1. 从 .twx 文件选择所需 BPM 集成服务来创建新模式实例。

    生成的 IBM Integration Bus 服务将包含一个服务接口（WSDL 文件），此接口具有一项其输入和输出与 BPM 集成服务定义相对应的操作。

  2. 开发 IBM Integration Bus 服务，将服务部署到集成节点（代理），并验证服务是否正确工作。
  3. 将服务导出为新的 .twx 文件。

    4. 新 .twx 文件以 BPM Web Service 集成的形式包含新服务的详细信息，其中包括服务 URL 和相应的业务对象映射。

最后，BPM 开发人员必须完成下列任务：

  1. 使用用于创建 BPM 集成服务的 BPM 工具箱，导入新 .twx 文件。

    2. 系统会用 IBM Integration Bus 服务的详细信息更新 BPM 集成服务。现在 BPM 集成服务可以调用 IBM Integration Bus 服务。
