BPM 統合

コンテキスト

IBM Business Process Manager (BPM) はエンタープライズ・レベルのビジネス・プロセスの設計とモデリングを提供し、IBM Integration Bus はエンタープライズ用の強力な接続と統合を提供します。

問題

ビジネス・アナリストが IBM BPM を使用してビジネス・プロセス設計 (BPD) を開発する場合、企業サービスから利用できる何らかのビジネス・データが必要になることがあります。 企業サービスの所有者は、必要なビジネス・データを取得するためにどのような情報を提供する必要があるかを、ビジネス・アナリストに知らせることができます。

ビジネス・アナリストは、BPD に必要な入出力ビジネス・オブジェクトを定義する方法は知っていますが、企業サービスへの接続を構成する方法は知りません。 例えば、企業サービスが特定のトランスポート・タイプを使用していたり、必要以上のデータを提供したりする可能性があります。また、セキュリティーが必要であったり、特定のサービス品質の順守を必要としたりする可能性もあります。

選択についてのガイダンス

このパターンは、IBM BPM ユーザーが BPM 統合サービスでインターフェースを定義済みで、そのインターフェースを IBM Integration Bus サービスを使用して実装する必要がある場合に使用します。
