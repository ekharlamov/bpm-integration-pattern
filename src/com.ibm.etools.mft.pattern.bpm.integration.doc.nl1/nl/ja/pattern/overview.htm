「Business Process Manager 統合サービス」パターン

このパターンを使用して、IBM Business Process Manager (BPM) 統合サービスと IBM Integration Bus サービスを統合できます。

ソリューション

このパターンを使用すると、IBM Business Process Manager から呼び出せる新規 IBM Integration Bus サービスを生成することができます。

このパターンを使用する前に、IBM Integration Bus インフォメーション・センターで IBM Business Process Manager に関する情報をお読みください (Business Process Manager との連動を参照)。

BPM 開発者と IBM Integration Bus 開発者が協働して以下のステップを実行して、IBM BPM 統合サービスと IBM Integration Bus サービスを統合します。

まず、BPM 開発者が次のタスクを完了する必要があります。

  1. BPM ツールキットを使用して、IBM Integration Bus サービスを呼び出す必要がある BPM 統合サービスを作成または特定します。
  2. 入力および出力の変数が定義された統合サービスが 1 つ以上含まれる BPM エクスポート (.twx ファイル) を作成します。

次に、IBM Integration Bus 開発者が次のタスクを完了する必要があります。

  1. .twx ファイルから必要な BPM 統合サービスを選択することによって、新規パターン・インスタンスを生成します。

    生成された IBM Integration Bus サービスには、BPM 統合サービス定義に対応した入力と出力を使用する操作を備えたサービス・インターフェース (WSDL ファイル) が格納されます。

  2. IBM Integration Bus サービスを開発し、そのサービスを統合ノード (ブローカー) にデプロイして、サービスが正常に機能することを確認します。
  3. サービスを新規 .twx ファイルとしてエクスポートします。

    4. 新規 .twx ファイルには、サービス URL と該当ビジネス・オブジェクトのマッピングが組み込まれた BPM Web サービス統合の実装の形式で新規サービスの詳細が格納されます。

最後に、BPM 開発者が次のタスクを完了する必要があります。

  1. BPM 統合サービスの作成に使用した BPM ツールキットを使用して、新規 .twx ファイルをインポートします。

    2. IBM Integration Bus サービスの詳細に基づいて BPM 統合サービスが更新されます。 これで、BPM 統合サービスが IBM Integration Bus サービスを呼び出せるようになりました。
