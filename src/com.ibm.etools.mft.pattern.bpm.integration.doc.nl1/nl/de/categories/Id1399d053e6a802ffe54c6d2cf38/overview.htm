BPM-Integration

Kontext

IBM Business Process Manager (BPM) ermöglicht die Entwicklung und Modellierung von Geschäftsprozessen auf Unternehmensebene, während IBM Integration Bus die für ein starkes Unternehmen erforderliche Konnektivität und Integration bereitstellt.

Problem

Wenn ein Unternehmensanalyst eine Geschäftsprozessdefinition mithilfe von IBM BPM entwickelt, benötigt er unter Umständen Geschäftsdaten, die über einen Unternehmensservice verfügbar sind. Der Eigner des Unternehmensservice kann dem Unternehmensanalysten mitteilen, welche Informationen zum Abruf der erforderlichen Geschäftsdaten bereitgestellt werden müssen.

Der Unternehmensanalyst weiß zwar, wie er die Ein- und Ausgabegeschäftsobjekte definieren muss, die für die Geschäftsprozessdefinition erforderlich sind, die Konfiguration der Konnektivität mit dem Unternehmensservice ist ihm jedoch nicht bekannt. Es ist beispielsweise denkbar, dass der Unternehmensservice einen bestimmten Transporttyp verwendet, mehr Daten als erforderlich bereitstellt oder die Einhaltung gewisser Sicherheitsmaßnahmen oder einer bestimmten Servicequalität verlangt.

Auswahlanleitung

Verwenden Sie dieses Muster, wenn ein Benutzer von IBM BPM eine Schnittstelle in einem BPM-Integrationsservice definiert hat und diese mithilfe eines IBM Integration Bus-Service implementiert werden muss.
