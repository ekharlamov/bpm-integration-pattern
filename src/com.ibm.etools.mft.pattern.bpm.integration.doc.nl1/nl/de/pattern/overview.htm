Muster 'BPM Integration Service'

Mit diesem Muster können Sie einen IBM Business Process Manager-Integrationsservice (BPM-Integrationsservice) mit einem IBM Integration Bus-Service integrieren.

Lösung

Mit diesem Muster können Sie einen neuen IBM Integration Bus-Service generieren, der von IBM Business Process Manager aufgerufen werden kann.

Lesen Sie vor Verwendung dieses Musters die Informationen zu IBM Business Process Manager im Information Center von IBM Integration Bus; siehe Mit Business Process Manager arbeiten

Die Entwickler von BPM und IBM Integration Bus arbeiten gemeinsam an der Integration eines IBM BPM-Integrationsservice in Verbindung mit einem IBM Integration Bus-Service, indem sie folgende Schritte ausführen:

Zuerst muss der BPM-Entwickler die folgenden Tasks ausführen:

  1. Er muss mithilfe eines BPM-Toolkits den BPM-Integrationsservice erstellen oder angeben, der den IBM Integration Bus-Service aufrufen muss.
  2. Er muss eine BPM-Exportdatei (Datei mit der Erweiterung .twx) erstellen, die mindestens einen Integrationsservice mit definierten Ein- und Ausgabevariablen enthält.

Anschließend muss der IBM Integration Bus-Entwickler die folgenden Tasks ausführen:

  1. Er muss eine neue Musterinstanz generieren, indem er den erforderlichen BPM-Integrationsservice in der Datei mit der Erweiterung .twx auswählt.

    Der generierte Integration Bus-Service enthält eine Serviceschnittstelle (WSDL-Datei) mit einer Operation, deren Eingaben und Ausgaben denjenigen der BPM-Integrationsservicedefinition entsprechen.

  2. Er muss den IBM Integration Bus-Service entwickeln, den Service auf einem Integrationsknoten (Broker) implementieren und prüfen, ob der Service ordnungsgemäß funktioniert.
  3. Er muss den Service als neue Datei (.twx) exportieren.

    4. Die neue Datei mit der Erweiterung .twx enthält Details zum neuen Service in Form einer Implementierung einer BPM-Web-Service-Integration, die die Service-URL und die entsprechende Geschäftsobjektzuordnung enthält.

Schließlich muss der BPM-Entwickler die folgenden Tasks ausführen:

  1. Er muss die neue Datei (.twx) mit dem BPM-Toolkit importieren, das für die Erstellung des BPM-Integrationsservice verwendet wurde.

    2. Der BPM-Integrationsservice wird aktualisiert und enthält jetzt die Details des IBM Integration Bus-Service. Der BPM-Integrationsservice kann jetzt den IBM Integration Bus-Service aufrufen.
